# 젠킨스 배포 버전 관리



## 롤백 ?

* 필요성 
  * 배포 한 후에 버그등의 이슈가 발생하여 이전 소스로 원복해야 하는 경우 사용
  * `GIT TAG`를 사용한다  (TGA:특정 시점의 스냅샷)

* 사용법
  * 개발 브랜치 배포
  * TAG를 남긴다 => 해당 시점 내용이 TAG로 저장
  * 이슈시, 직전에 저장된 TAG로 재 배포한다
